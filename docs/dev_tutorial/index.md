# 面向开发的教程

**本栏目旨在指引那些想要通过开发的方式来为启动器项目做贡献的人**（不是只有敲代码才叫做贡献）。

本章将尽量以最简单的方式来介绍开发的相关知识，并提供一些开发的参考资料。

**<mark>开发并不是那么简单的，也不是所有人都能适应开发工作。</mark>**

### 什么是编程

> 编程是把人类世界的问题，抽象成计算机世界的逻辑，并指挥计算机去执行。

### 学习编程必需的能力

1. 搜索引擎的使用：主流搜索引擎见 2.3 使用搜索引擎查找资料；
2. 寻找官方网站，注册账号、下载正版软件的能力；
3. 基本的英语水平，要理解报错信息的内容；
4. 合理提出问题的能力，详见[编程交流群生存指南](https://mdr-c-tutorial.github.io/C/%E6%9D%82%E9%A1%B9/%E6%8A%80%E6%9C%AF%E6%97%A0%E5%85%B3/1_%E7%BC%96%E7%A8%8B%E4%BA%A4%E6%B5%81%E7%BE%A4%E7%94%9F%E5%AD%98%E6%8C%87%E5%8D%97)；
5. 理解语言的能力：
    1. 编程语言是人写的，也是给人看的，编程语言当中有自然语言的影子；
    2. 把你的代码写得更像自然语言，就能自然地契合“如无必要，勿添注释”的思想；
    3. 函数的名称有意义，了解这些名称（的词源）有助于记忆函数的功能，也方便在正确的使用场景想起特定函数。

### 如果你过度依赖 Vibe Coding / AI 进行开发

首先你要知道，Vibe Coding 是什么？
> Vibe Coding 是一种由 AI 驱动的沉浸式编程模式，核心理念是只关注结果，不关心实现细节。开发者通过自然语言、示例或图形描述需求，AI 自动生成代码并运行，用户根据结果即时反馈，AI 再进行调整，形成 “提出想法 → 查看结果 → 调整 → 再生成” 的高效闭环。

> 这种方式下，人类的角色是出题与审核，AI 负责解题与改进。相比传统编程需要手写逻辑、调试 bug，Vibe Coding 让开发者全程沉浸在创意与验证中，极大提升效率。

说白了就是，你跟 AI 说需求，它给你出成品。但是，大型项目万万不可取的是过度依赖 AI。**Vibe 的作用是减少程序员工作的时间，而不是降低编程的门槛**。但是现在很多没有任何编程基础的人试图通过 Vibe 来编程。如果你想为了做贡献而使用 AI，那你还是算了吧。

**过度依赖 AI 的后果有：**
1. 钱包空空（普通的对话 AI 根本不适合 Vibe，只有接入工作环境的 Agent 才能 Vibe，然而 Agent 是要花钱的）。
2. 你不会有任何的代码理解或者编写能力，甚至还会退化。
3. 代码质量显著偏低，且要花两倍或更高的成本来给 AI 擦屁股。
